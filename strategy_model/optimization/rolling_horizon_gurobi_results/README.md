## Important Note About Old 33.16% Number

An earlier CAES 100 MW / 24h run produced:

```text
33.16% improvement
```

That was before applying the stricter PNNL DoD setup with:

```text
Cmin = 20% of capacity
initial SoC = midpoint of allowed range
```

The corrected number to use for presentations and current reporting is:

```text
32.33% improvement
```

## Large Files

The full generated label/checkpoint CSVs are intentionally not tracked in Git because they are very large and GitHub rejects files over 100 MB.

The summary files are tracked. The full labels can be regenerated by rerunning:

```bash
./venv/bin/python strategy_model/optimization/rolling_horizon_gurobi_dispatch.py \
  --horizon-hours 168 \
  --step-hours 24 \
  --storage-type caes \
  --storage-rating 100 \
  --storage-duration 24 \
  --min-soc-frac 0.2 \
  --max-soc-frac 1.0 \
  --initial-soc 1440 \
  --out-dir strategy_model/optimization/rolling_horizon_gurobi_results/full_dataset_caes_100mw_24h_dod80_mid_soc
```
